# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it by creating a [GitHub Issue](../../issues).

### Guidelines for Reporting:
- Clearly describe the vulnerability.
- Include steps to reproduce the issue, if possible.
- Avoid sharing sensitive details publicly. If the issue requires confidentiality, please let us know, and we can discuss alternatives.

We will:
1. Acknowledge your report within **48 hours**.
2. Investigate and address the issue promptly.
3. Provide updates on our progress through the issue tracker.

Thank you for helping to make this project more secure!
